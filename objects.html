Objects

Java is known as an Object-Oriented programming Language. This means many of its functions are based off containers called objects, or (in the case of Java), classes. This offers many advantages to developers, but the most important is less repetition when coding. Creating objects allows a programmer to store categorized data and reuse sections of code without having to retype it somewhere else in the program, and is therefore a much more efficient and streamlined way to program.

Classes

A class keyword in Java denotes an object. Objects are used to group methods and variables that have a logical place with one another and will interact in some way with one another. If we have a Car, we want it to be able to drive and stop. If we take that last sentence and look at it a little bit harder, we can already see our class taking shape.

Declaring a Class

Before a class can be used, it must be declared inside its own .java file. In this exmple, we will create a Car class that can drive and stop. We will also include the necessary variables needed to make (construct) a car.

Class Header

public class Car {

A class header is used to declare your class to the program. It is a very simple declaration including only an accessibility identifier, the word class, and the name of the class you would like to create. A Class's name usually starts with a capital letter. (A syntax error will not be thrown if you don't capitalize it, but it is the convention of programmers).

Constructors

All classes should contain a constructor. The constructor, as it might sound, constructs a new instance of that object. A constructor is most-often used to create a new instance of the object outside of the class, but methods within the object's class can also call its constructor and create new instances (usually to return the objects to another class).

Structure of the Constructor

public <Name of class>() {

A constructor is normally used to initialize uninitialized variables. For example, in our car class, we have two instance variables, topSpeed and sixCylinder. In our class, sixCylinder is already initialized as false, but topSpeed has no initial value. In this case, our constructor would be used to initialize it.

The completed Constructor would be as follows:

public Car(int top) {
     this.topSpeed = top;
}

You can see the assignment statement inside the constructor. The constructor works just like any other method in Java, but the computer will recognize a call to a constructor as an instruction to make a new version of that object, and allocate it some memory in RAM.
