Variables

Variables are one of the most important pieces of any programming language, and even more so in an object-oriented language.

Working with Variables

Variables can be used to store any kind of data that a program can work with. In addition to primitive types, variables can contain other, developer-created objects. Their use ranges from simple data storage to collecting and interpreting user input.

Declaring a Variable

Before a variable can be used, it must be declared. Declaring variables in Java is very easy. The following is an example declaration of a variable x that contains a whole number:

int x;

That's it!
In Java, this small statement tells the computer that x will always contain an Integer, or int. As a general rule, Once a variable's type has been declared, it may not be changed.

Initializing / Assigning A Variable

Obviously, a variable without any value really has no purpose, and simply declaring a variable does not give it a usable value (though there are default values for some datatypes). To fix this, we must initialize a variable. Initialization (or assignation) gives the variable its value and prepares it to be used in the program. This is most often done with the use of a constructor or initializer.

To initialize a variable, one must make use of the assignment operator (=). The assignment operator tells the computer to assign a value of any type to a variable of the same type. If we wanted to initialize an int and a String, we would simply write statements like so:

int x = 5;
String s = "cookies";

If you look carefully, the above statements are actually both declaration and initialization statements combined into one. int x being the declaration and = 5 being the assignment or initialization. The same is true with String s = "cookies"

Types of variables

Primitive Types

There are infinitely many types of Variables, but the few that you should be most familiar with are the primitive types. There are a few primitive types, but the ones that will arise most often include:

  • int                 //Whole Numbers (Integers)
  • double or float     //Positive/Negative Numbers containing a decimal point
  • boolean             //True or False
  • char                //A single letter or symbol (character)
  • String              //Whole words or phrases, surrounded by quotes ("") or ('') (Not actually a primitive type since it has methods that can be used to perform tasks, but acts as one, so we'll place it here since it is very important)

    • Each of these types represents a different form that a variable can take, and that are built-in to the Java language by default.

    Objects

    Any classes created in the program can be initialized and stored inside a variable. For example, say we have a Game class, and every time the player clicks a button on screen, we would like to create a new instance of the Game to be played.

    To assign a new instance of the Game class to a variable, we would simply type

    Game myGame = new Game();

    We would call this new variable a Game object, because it is now its own seperate instance of the Game class. Whatever modifications are made to myGame will not affect any other instances of the Game class. If Game had a non-static method getCurrentPlayers(), we would be able to call this method by coding myGame.getCurrentPlayers(); In fact, if this method were static, this call would not work, throwing a Syntax error. The Game(); part is simply shorthand for calling the Game class's constructor method, which in this case has no parameters.

    Constants

    A constant is only useful when presenting a value that will never change throughout the life of your program. Since constants never change, they are a great way to ensure that no value can be accidentally modified, later causing logic or runtime errors. Possible uses of constants include length conversions and mathematical constants like e or pi.

    Declaring constants is easy. Before the type, simply type the keyword final. The Computer will see this and know that you have declared a constant rather than a variable, and will throw errors if a change is attempted later in the program.
