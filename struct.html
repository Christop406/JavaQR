Structure

Java is a Strict Type Language. This means every variable, method and class has to be explicitly defined a certain type.

There are some different Primitve Types within Java. A primitive type is a type of variable that does not conform to any interface or class and therefore cannot make use of any methods.

Structure of a Java Program

Classes

Classes, otherwise known as objects, are containers for methods and variables that have a close relationship to one another in a way that makes sense to the program and other programmers.
Every method and variable in Java must be encapsuled by a class or interface.

Methods

Methods allow a programmer to embed repeated or lengthy amounts of code inside a function that exectues when it is called. Methods can do everything from a simple math calculation to large algorithms to displaying colors on a computer screen. The only limit to a method's actions is the programmer's imagination.

Variables

Variables store data in the program. When a variable is declared, it gets a little spot in the computer's RAM and keeps that variable until it is no longer needed or reset.

Basic Structure of a Java Class

A Basic Java class consists of three parts:

  • The Accessibility Identifier (public or private)
  • The class declaration.
  • The name of the class <name>

    Structure of Java Methods

    Java's methods all share a similar design pattern to one another. To declare a method, one must include:

  • An Accessibility Identifier
  • A return type. A return type can be any object or primitive type.
  • A method name: exampleMethod in the example below.
  • Input parameters surrounded by parentheses. (int input)
    • public int exampleMethod(int input){
         return input;
    }

    The above code takes in an int (named input) as an input parameter. It returns the number that was passed in by the method call.

    Control Flow

    Control Flow Statements are the built-in Java functions that check if an expression is true or false. These statements include if and switch statements, as well as loops.

    if statements

    if statements serve only one purpose: To check if an expression is true or false. If the expression returns true, the code inside the if statement will execute. If not, the program will simply pass over the code, not executing it.

    Structure of an if statement

    if(someExpression){
    ...code...
    }

    If someExpression evaluated as true, the program would execute the code within. If not, the program would not.

    Inside an if statement can be an expression using logical operators OR a boolean variable representing true or false.

    For example, if I were to check if two int variables were equal, I could write it in two ways:

    int x = 5;
    int y = 7;
    if(x == y){
         ...code
    }

    OR

    boolean xAndY = (x == y)
    if(xAndY){
         ...code
    }

    if statements are applicable to all aspects of programming and are one of the most important tools a programmer can have when making programs. Without if statements, programs simply would not work, or at least not be very complex.

    while loops

    while loops are most-often used to repeatedly execute blocks of code an uncertain amount of times during the program's run cycle. while and for loops are very similar, and are often used purely at the discretion of the programmer, since they can both do about the same thing in nearly the same way.

    Any of Java's loops contains three necessary parts. A start index, an index variable, and an end index. In while loops, the start and end indexes will usually take the form of variables, rather than integer or declared constants, to comply with a while loop's "open-ended" nature.

    int x = 0;     //Setting our index variable 'x' to our start index 0
    while(x < 10){ //Loop for as long as x is less than 10
         ...code;  //The Programmer's Code
         x++;      //Adding 1 to the index variable
    }

    As you can see above, the start index and index variable must be declared outside the while loop, but incremented inside the loop. The incrementation helps to avoid the problem of an infinite loop, where a loop never stops executing. This will crash a program.

    for loops

    A for loop is quite like a simplified while loop. This means that the declaration for one may be more compact than that of a while loop, but is more concrete and not as easy to change throughout the run of the loop.

    for(int i = 0; i < 10; i++){
         ...code
    }

    A for loop's declaration is formatted as follows: for(<start index>;<end index>;<incrementation>){

    Most often, the <start index> contains a variable declaration and initialization, like in the above example. This makes it easier to setup a variable, its start and end indexes, and increment it all in one line. The last statement, i++, simply means: "add one to the index variable i every time the code within ends." To add more than one to the index variable, one might use i = i + <some number> or i += <some number>. Any variable may be used in place of i.

    Logical Operators

    Logical operators are the main method by which a programmer can control what parts of code are executed and when they are executed. These operators are usually found within if, while, for, and switch statements.

    EQUALS (==)

    The EQUALS operator is used to compare the values between two variables or constants of a primitive type (not String). The EQUALS operator evaluates to true or false, meaning it can be used along with any of the other logical operators and inside any of the control flow statements.

    (5 == 7)   // evaluates to false
    (10 == 10) // evaluates to true

    The computer views these expressions exactly as it would true or false, and can be assigned to any variable as such: boolean isItTrue = (5 == 5);

    AND (&&)

    The Logical AND operator is used to compare two or more different boolean expressions. The AND operator is only true if both conditions also evaluate to true. If even one of the expressions evaluates to false, the whole expression evaluates to false.

    For example, a truth table (useful for deciding how expressions will evaluate when added together):

    (false && true) = false
    (false && false) = false
    (true && true) = true

    OR (||)

    The Logical OR operator checks for the truth of one expression. This means, if even one of the evaluated expressions evaluates to true, the whole statement is true.

    (true || false) = true
    (false || false) = false

    NOT (!)

    The Logical NOT operator can be used with any of the other logical operators. It is used to get the opposite boolean value that the expression evaluates to. The NOT ! operator must enclose the whole boolean expression OR Can be used to NOT a single boolean variable. Uses could look like this:

    !(true && true)             // Evaluates to false
    !false                      // Evaluates to true
    (false || (true && !false)) // Evaluates to true
    NOT EQUAL (!=)

    The NOT EQUAL operator is simply the EQUALS operator, but NOTted. You use it to find the existence of a difference between multiple values.
