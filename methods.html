Methods

Methods serve as containers for code that will be repeated within the rest of the program. Also known as functions, they play an integral role in Object-Oriented Programming, and all languages use them.

Detailed structure of a method header

When beginning to craft a method, it is important that you know what the method will be doing with the data and plan its structure accordingly. As an example, we will create a method that takes in a number and adds 20 to that number, then returns it back to the main program.

But How do we Start? Well, first, let's pseudocode what we want the method to do and what we will need to make that happen.

//Take the number the sender gives to the method... I think it will be an int.
//Add 20 to that number
//Return our new number. We'll call it newNumber


Now that you've sucessfully pseudocoded the new method, it's time to implement it.

//Take the number the sender gives to the method turns into public int methodAddsTwenty(int num){

Likewise, //Add 20 turns into newNum = num + 20;

Finally, the return statement looks like this: return newNum;

Then we add our closing semicolon: } and we're done! If we were to call this method using a variable assignation, it would look something like this:
int number = methodAddsTwenty(5);
After the call, number would equal 25, because the value returned by methodAddsTwenty(5) gets automatically assigned to the variable.

There are four parts to a method header. Two of which are always required, and the other two can be ommitted based on the function of the method or if only default values are required.

A general method header might looks something like this:

<accessibility> <returntype> <name> (<parameters>) {
     //code
}

<accessibilty> denotes which parts of the program can access and use this method. When set to private, only other methods in this class may access the method. When set to public, other classes and other parts of the program may use the method when they instantiate the class at runtime.

<returntype> denotes the type of the value to be returned. This can be any type acessible to the method.

<name> presents an identifier that the rest of the class and program will use when calling the method.

<parameters> are comma-separated lists of variables or constants that can be used to modify the run of the method, and are required when calling the method. If the above method's parameters were (int num, String s), the method would need to be called as <name>(5, "puppies") to avoid a runtime and/or syntax error.

Parameters

Parameters are important when dealing with methods. Simply put, they can be seen as modifiers to how a method executes by passing in different values.
For Example:

doesThis(String s), doesThat(int x), doesTheOtherThing(int x, String s).

All of these methods may or may not do the same thing, but they are completely different (to a computer, anyway) in their input parameters. When declaring a method, the input parameters must contain a type and a name. You see this above with String and int, s and x, respectively. When calling a method containing input parameters, a type need not be specified, as the computer will infer its type when the method is called. For example, when calling doesThis, one need not type doesThis(String "this"), because the double quotes (") denote a String to the Java compiler. An int will be inferred by simply typing in any number (doesThat(10)). Just because type declarations are not required, however, does not mean the computer is lenient on what is passed in as a parameter. A method call with the correct name but incorrect parameters results in a syntax error, and the program will not execute.

Return Types

A return type can be any type of variable accessible to the method, or void.

Method Calls

When a program wants to use a method. It performs what is called a method call. A method call tells the computer which method it would like to run, including its input parameters, and usually assigns this value to a variable.

A method call is structured as follows:

myMethodName(inputParameter);

When assigned to a variable, if the method has a return type, whatever the method returns will be assigned to the variable in the other class.

Static methods

Sometimes you will want to use methods in your program without having to make an instance of the object containing the method. For this reason, Java includes what are called static methods. These methods are not called from an instance of their class. Instead, the class acts as more of a "container" for the method.

Suppose a class has been created with the name School. Inside the class, we want to include a description of a general English class. In this case, we might use a static method, because we do not need to create a new School to know what an English class is. Our method will return a String that describes an English class.

To Create this method, we would add to our class:

public static String englishInfo(){
     return "A class where the English language is Learned";
}

When calling this method, we would now write
School.englishInfo()
rather than
School s = new School();
s.englishInfo();
